| SaSp01 | 20.05.2009 | 40 % | tolerierbar |
| Beschreibung des Risikos: Aufgrund des potenziell großen Umfangs einer einzelnen Situation, kann eine zu hohe Barriere zum Eintritt eintreten, was zu einer geringeren Nutzung der Funktion von benutzerdefinierten Situationen führen kann. |  |  |  |
| Kontext / Details |  |  |  |
| Einzelne Situationen können schnell ein grosses Ausmass annehmen, damit Schüler Spaß daran haben zu lernen um in den Leaderboards aufzusteigen.  |  |  |  |
| Überwachung: Wie kann das Risiko überwacht werden?  Es werden monatliche Reports generiert, welche die Verhältnisse zwischen der Nutzung von benutzererstellten Situationen zu zuvor erstellten Situationen. Sollte das Verhältnis nach 2-3 Monaten unter 20% liegen, werden Mitarbeiter informiert. |  |  |  |
| Vermeidung/Verringerung Was kann die Wahrscheinlichkeit für das Auftreten des Risikos vermieden oder verringert werden? Wie kann der (erwartete) Schaden verringert werden?  Situationen selbst können angepasst werden, es sollte sowohl ausführliche als auch kurze Situationen geben. Erstellung eigener Situationen sollte möglichst intuitiv und simpel sein. Werden Mitarbeiter informiert, dass eine Schule unter dem Verhältnis liegt, sollte der Kontakt mit der Schule gesucht werden, um Feedback einzuholen und um potenziell Schulungen anzubieten, wie eigene Situationen erstellt werden können. |  |  |  |
| Management / Reaktion  Können schon jetzt Maßnahmen ergriffen oder vorgesehen werden, für den Fall, dass das Risiko eintritt? Maßnahmen zur Schadensbegrenzung (Begrenzung der Auswirkungen) Mit welchen Maßnahmen kann das Ausmaß des Schadens verringert werden?  Optionale Tutorials, bei denen ausgewählt werden kann, wie sehr das Tutorial ins Detail gehen soll, damit jeder die Prozedur so gut es geht verstehen kann. Anbietung von Schulungen mit Live-Demonstrationen, wie Situationen erstellt werden, an denen Lehrer teilnehmen können. Eingebaute KI ins Admin-Center, welches sowohl Pointer zur eigenen Situationserstellung geben kann, als auch selbst gesamte Situationen generieren kann.   |  |  |  |

| SaSp02 | 20.05.2009 | 80 % | katastrophal |
| Beschreibung des Risikos: Aufgrund der Entwicklung der App ausschließlich für Android Geräte, kann ein großer potenzieller Nutzeranteil abgeschnitten werden, wodurch sich die aktiven Nutzerzahlen stark verringern würden. |  |  |  |

| Viele Schüler und Schulen verwenden zum Großteil IPads / Apple Beamer, wodurch die Nutzung der App, welche aktuell nur für Android entwickelt wird, stark erschwert wird. |  |  |  |
| Überwachung: Wie kann das Risiko überwacht werden?  Umfragen an Schulen und Interviews mit Lehrern durchführen, um zu erfahren wie das Verhältnis zwischen Nutzung von Apple und Android Geräten an weiterführenden Schulen aussieht |  |  |  |
| Vermeidung/Verringerung Was kann die Wahrscheinlichkeit für das Auftreten des Risikos vermieden oder verringert werden? Wie kann der (erwartete) Schaden verringert werden? Um den Apple App-Store zu umgehen, müssten Wege recherchiert werden, mit denen man Software auf IPads ohne den Store installieren kann. Wenn dies realistisch erscheint, könnten Schulen die App ebenfalls manuell an Schüler verteilen, die diese haben wollen. |  |  |  |
