Zweck des Dokumentes 'Projektvision'

(Zielgruppe, Zweck, Inhalt)

Die Projektvision ist eine Beschreibung der Projektidee, um Geldgeber, Mitstreiter, Fürsprecher für ein Projekt zu finden.

Diese Projektidee/Projektvision beschreibt das System in drei Zeithorizonten:

* Für die erste Auslieferungsversion  
* Für die weiteren Auslieferungsversionen (Releases)  
* langfristig, visionär

| Zeitpunkt von Beginn und Fertigstellung | für Budgetgespräche, um Ressourcen einzuwerben zur Vertriebsfreigabe, um Werbung für das Produkt zu machen  |

| 1 | Das erstes Release (Erste Kundenversion)  Wie sieht die erste Version aus, die an den Kunden geliefert wird?  |  |
| :---- | :---- | :---- |
|  | Projektbeschreibung (initial) | Das erste Release von **SafetySpot** stellt eine funktionsfähige, aber reduzierte Erstversion der mobilen Lern-App dar. Ziel dieser Version ist es, Schulen erstmals ein digitales Werkzeug zur Verfügung zu stellen, mit dem Schüler spielerisch für Gefahrensituationen im Schulalltag sensibilisiert werden können. Die Anwendung konzentriert sich auf ausgewählte typische Gefahrensituationen (z. B. Chemieraum, Werkraum, Sportunterricht, Straße) und ermöglicht es Schülern, diese in Form von Szenarien zu bearbeiten. Lehrkräfte erhalten einen einfachen Admin-Bereich, um Klassen anzulegen, Inhalte zu verwalten und den Lernfortschritt einzusehen.Das erste Release ist als **Prototyp mit echtem Mehrwert** gedacht, der bereits im Unterricht eingesetzt werden kann.. |
|  | Projektziele (initial) | Was sind die Ziele für das erste Release? |
|  |  | Die Ziele des ersten Releases sind: Entwicklung einer lauffähigen Android-App zur Gefahrenaufklärung im Schulkontext Vermittlung grundlegender Sicherheitsregeln durch interaktive Szenarien Erhöhung der Motivation der Schüler durch spielerische Elemente (Punkte, Rankings) Bereitstellung eines einfachen Admin-Bereichs für Lehrkräfte Ermöglichung einer klassenbasierten Nutzung in Schulen Gewinnung erster praktischer Erfahrungen mit digitalen Lernsystemen   |
|  | Anwendungsszenarien(initial) | Welche Anwendungsszenarien werden mit der ersten Version bedient?  **Szenario 1: Schüler lernt Gefahrensituationen** Ein Schüler meldet sich in der App an, wählt die Kategorie „Chemie“ und spielt ein Szenario zum sicheren Umgang mit Chemikalien. Nach jeder Aufgabe erhält er direkt Feedback und sammelt Punkte. **Szenario 2: Lehrer überprüft Lernfortschritt** Eine Lehrkraft meldet sich im Admin-Bereich an, sieht die Ergebnisse ihrer Klasse und erkennt, bei welchen Szenarien häufig Fehler auftreten. So kann sie den Unterricht gezielt anpassen. **Szenario 3: Wettbewerb innerhalb einer Klasse** Eine Klasse nutzt SafetySpot über mehrere Wochen. Die Schüler vergleichen ihre Punktestände im Klassen-Leaderboard, was die Motivation zur wiederholten Nutzung erhöht.  |
|  | Qualitätseigenschaften | Welche Qualitätsziele sollen bei der ersten Version erfüllt werden? Für die erste Version stehen folgende Qualitätsziele im Vordergrund: **Benutzbarkeit (hoch)** Die App muss einfach und intuitiv bedienbar sein, insbesondere für Lehrkräfte ohne technische Vorkenntnisse. **Zuverlässigkeit (mittel)** Die Kernfunktionen (Login, Szenarien spielen, Fortschritt speichern) müssen stabil funktionieren. **Sicherheit (grundlegend)** Zugriff erfolgt über Benutzername und Passwort, Datenübertragung erfolgt geschützt. **Performance (ausreichend)** Inhalte sollen schnell laden, auch auf älteren Geräten.  |

| **3\. Vision** Wo will man in 3,5 oder 10 Jahren sein? |  |  |
|  | Projektbeschreibung (langfristige Vision) | Langfristig soll SafetySpot eine **etablierte digitale Lernplattform für Sicherheits- und Gefahrenprävention** im Bildungsbereich werden. Die App soll nicht nur im Schulalltag, sondern auch in anderen Bildungskontexten (z. B. Berufsschulen, Ausbildungsbetriebe) eingesetzt werden. SafetySpot entwickelt sich von einer einfachen Lern-App zu einem umfassenden Präventionssystem, das digitales Lernen, Gamification und realitätsnahe Simulationen verbindet.  |
|  | Visionäre Projektziele (langfristige Vision) | **Pädagogische Ziele:** Nachhaltige Erhöhung des Sicherheitsbewusstseins bei Kindern und Jugendlichen Verankerung von Gefahrenprävention im Schulalltag Lernen durch realitätsnahe Simulation statt reiner Theorie **Betriebswirtschaftliche Ziele:** Etablierung von SafetySpot als lizenzierbare Plattform für Bildungseinrichtungen Kooperationen mit Schulträgern und öffentlichen Institutionen Aufbau eines skalierbaren Geschäftsmodells im EdTech-Bereich  |
|  | Anwendungsszenarien (langfristige Vision) | Schüler trainieren Gefahrensituationen in interaktiven Simulationen (z. B. VR/AR-Module) Schulen nutzen SafetySpot als festen Bestandteil ihres Sicherheitskonzepts Lehrkräfte erstellen und teilen Szenarien über eine zentrale Plattform Behörden oder Institutionen nutzen anonymisierte Daten zur Verbesserung von Sicherheitskonzepten  |

|    | Was sind die Ziele? | In diesem Dokument sollen die (Top-)Ziele beschrieben werden, die mit dem Projekt verfolgt werden. Top-Ziele sind dabei grundsätzlich bei den Geldgebern zu suchen. Mögliche Top-Ziele im operativen Betrieb sind: Vermarktung Rationalisierungseffekte Erhöhung der Kundenbindung Mögliche Ziele in Forschung & Entwicklung sind: Erfahrung mit einer Technologie gewinnen Auswahl einer Technologie Es soll ein Dokument vorgestellt werden, in dem die Ziele beschrieben werden: Das Ziel, welches implementiert wird, aus der Sicht des Anwenders – mit dem Mehrwert. Die 'Vision' , wie die App sich weiterentwickeln könnte oder in der Endausbaustufe aussehen könnte.  |
|  | Was sind Anwendungs-szenarien? | Als Unterziel spielt dann oftmals der 'Mehrwert' für die Nutzer eine Rolle:  Angebot toller Feature in der App um eine Vermarktung zu erleichtern. Dieser Mehrwert sollte ebenfalls beschrieben werden. Zur Beschreibung sollte die Ebene der 'User Stories‘ gewählt werden. Für das Schreiben einer User Story kann es hilfreich sein, auf das folgende Grundgerüst zurückzugreifen: Als (Anwendertyp) möchte ich (folgende Aktion durchführen), um (dieses Ziel zu erreichen). |
